# TanStack Query patterns

`createConfigQuery` returns a standard TanStack `queryOptions` object.
Every pattern that works with `useQuery` works here.

## `useSuspenseQuery` — Suspense-driven

`data` is always defined when the component renders.
Wrap with `<Suspense>` and `<ErrorBoundary>` to handle loading and errors declaratively.

```tsx
const { data: config } = useSuspenseQuery(appConfigQuery);
return <p>{config.apiUrl}</p>;
```

<Canvas of={Stories.Suspense} />

## `useQuery` — inline loading states

No `<Suspense>` or `<ErrorBoundary>` needed.
Handle loading and errors inline, just like any `useQuery` call.

```tsx
const { data: config, isLoading, isError, error } = useQuery(appConfigQuery);
if (isLoading) return <p>Loading…</p>;
if (isError)   return <p>Error: {String(error)}</p>;
return <p>{config!.apiUrl}</p>;
```

## Retry

Use TanStack's built-in `refetch()` or `invalidateQueries()` — no custom retry logic needed.

```tsx
const { isError, refetch } = useQuery(appConfigQuery);
// or
queryClient.invalidateQueries(appConfigQuery);
```
