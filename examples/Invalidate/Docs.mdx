# Invalidating config

Config is cached indefinitely ‚Äî it loads once and never re-fetches on its own.
To force a fresh load, invalidate the cache.

## Inside a component ‚Äî `useConfigInvalidate`

```ts
const { invalidate, isPending, isError, error } = useConfigInvalidate(appConfigLoader);

invalidate();   // marks stale ‚Üí triggers a re-fetch
isPending       // true while re-fetching
isError         // true if the new load failed
error           // the ConfigPipelineError, if any
```

## Outside a component ‚Äî `queryClient` directly

Use `loader.queryOptions()` to target a specific cache entry, or `loader.queryKey()`
for prefix-matching to invalidate all variants at once:

```ts
// specific entry
await queryClient.invalidateQueries(appConfigLoader.queryOptions());

// all variants of a context-aware loader (prefix match)
await queryClient.invalidateQueries({ queryKey: todoLoader.queryKey() });
```

---

## With `useConfigQuery` ‚Äî inline error state

The simplest approach. `isError` / `isLoading` / `data` are handled inline ‚Äî
no boundary to reset.

```tsx
const { data, isError, error } = useConfigQuery(appConfigLoader);
const { invalidate } = useConfigInvalidate(appConfigLoader);

{isError && (
  <button onClick={invalidate}>Retry</button>
)}
```

## With `useConfigSuspenseQuery` ‚Äî `QueryErrorResetBoundary`

When using Suspense, an `<ErrorBoundary>` stays in its error state even after
the config recovers ‚Äî it never sees the successful re-fetch because it stopped
rendering its children.

The fix is `QueryErrorResetBoundary` from TanStack Query. It provides a `reset`
function that clears TanStack's internal error state. Wire it to the boundary's
`onReset` so both reset together when the user retries.

```tsx
import { QueryErrorResetBoundary } from '@tanstack/react-query';

<QueryErrorResetBoundary>
  {({ reset }) => (
    <ErrorBoundary
      onReset={reset}
      fallback={(error, resetBoundary) => (
        <div>
          <p>{error.message}</p>
          <button onClick={resetBoundary}>Retry</button>
        </div>
      )}
    >
      <Suspense fallback={<p>Loading‚Ä¶</p>}>
        <ConfigDisplay />
      </Suspense>
    </ErrorBoundary>
  )}
</QueryErrorResetBoundary>
```

`resetBoundary()` calls `onReset` (which clears TanStack's error state) and
then clears the boundary's own error state. The boundary remounts its children,
the query re-fetches, and the Suspense fallback shows while loading.
